# AI 助手浮动窗口使用说明

## 🎉 功能更新

AI 助手现在已经升级为**全局浮动窗口**模式，具有以下新特性：

### ✨ 新功能特点

1. **全局可访问**
   - AI 助手现在是一个浮动按钮，始终显示在学生界面的右下角
   - 可以在任何页面（论坛、资源浏览、个人中心等）随时点击打开
   - 不再需要切换到特定的"AI 助手"页面

2. **聊天记录持久化**
   - 聊天记录自动保存到本地浏览器（localStorage）
   - 切换到其他页面后再回来，聊天记录依然保存
   - 关闭浏览器后重新打开，聊天记录仍然存在
   - 只有手动点击"清空聊天"按钮才会清除记录

3. **优雅的 UI 设计**
   - 浮动按钮带有渐变背景和动画效果
   - 聊天窗口支持平滑展开/收起动画
   - 响应式设计，支持移动端

## 📱 使用方法

### 打开 AI 助手
1. 登录学生账号
2. 在学习中心的任何页面，都能看到右下角的"🤖 AI 助手"浮动按钮
3. 点击按钮即可打开聊天窗口

### 发送消息
1. 在输入框中输入问题
2. 按回车键或点击"发送"按钮
3. 等待 AI 回复（显示加载动画）

### 清空聊天记录
- 点击聊天窗口顶部的"🗑️"按钮
- 确认清空后，聊天记录将被清除并重置为初始问候语

### 关闭聊天窗口
- 点击右上角的"✕"按钮
- 或直接点击浮动按钮切换开关状态

## 🔧 技术实现

### 文件结构
```
src/
├── stores/
│   └── aiAssistant.ts          # Pinia 状态管理
├── components/student/
│   └── AIAssistantFloating.vue # 浮动窗口组件
└── layouts/
    └── StudentLayout.vue       # 学生布局（已集成 AI 助手）
```

### 数据存储
- **存储位置**: localStorage
- **存储键**: `ai_messages`
- **数据格式**: JSON 数组
- **包含内容**: 消息角色、内容、时间戳

### 状态管理
使用 Pinia 进行全局状态管理：
- `isOpen`: 窗口开关状态
- `isLoading`: 加载状态
- `messages`: 消息列表
- `addUserMessage()`: 添加用户消息
- `addAIMessage()`: 添加 AI 回复
- `clearMessages()`: 清空聊天记录
- `toggleOpen()`: 切换窗口开关

## 🎨 样式定制

如果需要修改 AI 助手的样式，可以在 `AIAssistantFloating.vue` 中调整：

### 修改浮动按钮颜色
```css
.float-button {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  /* 修改这里的颜色值 */
}
```

### 修改窗口大小
```css
.chat-window {
  width: 380px;    /* 窗口宽度 */
  height: 550px;   /* 窗口高度 */
}
```

### 修改窗口位置
```css
.ai-assistant-container {
  bottom: 20px;    /* 距离底部距离 */
  right: 20px;     /* 距离右侧距离 */
}
```

## 📝 注意事项

1. **聊天记录**
   - 聊天记录仅保存在当前浏览器
   - 更换浏览器或清除缓存会导致记录丢失
   - 建议定期备份重要对话

2. **网络要求**
   - 需要后端服务运行才能调用 AI
   - 确保 `.env` 文件中配置了正确的 API Key

3. **性能**
   - 聊天记录过多可能影响性能
   - 建议定期清空不需要的记录

## 🐛 故障排除

### AI 助手按钮不显示
- 检查是否登录了学生账号
- 刷新页面
- 检查浏览器控制台是否有错误

### 聊天记录丢失
- 检查是否清除了浏览器缓存
- 确认使用的是同一台电脑和浏览器

### AI 不回复
- 检查后端服务是否运行
- 检查 API Key 配置是否正确
- 查看浏览器控制台的错误信息
